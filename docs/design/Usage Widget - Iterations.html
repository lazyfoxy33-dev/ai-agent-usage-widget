调和品牌底色 + 紧凑横排

保留每家 provider 的识别色，但把它收敛成强调色而非整块面板底色，让三端读起来像一个组件。在此之上叠加语义告急色，并把双倒计时合并为「最早重置者」。

01

核心抉择：环 vs 条

同一份数据、同一套调和底色。环信息密度高、桌面 Widget 上更精致，中心数字改为显示「最告急的窗口」消除歧义；条对「我有多满」的判断最直接，且能无损缩放到 Touchbar。

语义告急色 实战

同一条量表在不同已用量下的取色。平时是品牌色，≥70% 转琥珀、≥90% 转红——一眼分辨「还早 / 该省着用 / 快爆了」。

05

面向未来：充值消耗型 provider

订阅制是「窗口已用 % + 重置倒计时」；充值制是「余额烧完 % + 按当前速率的续航估算」。同一行结构直接复用——单条量表 + 续航文案替代重置倒计时，无需为新形态另起炉灶。
